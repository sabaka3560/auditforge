🔗
matcher.py
engine/matcher.py
+
4-stage waterfall header matcher. Given ideal column names and actual file headers, returns one MappingResult per ideal column. First hit wins; Unmatched if all 4 fail.
Main function
build_mapping(ideal_rows, actual_headers, threshold=80) → list[MappingResult]
4-Stage waterfall (first hit wins)
① Exact — ideal_name in actual_headers
② Normalized — strip_header(ideal) == strip_header(actual)
③ ManualAlias — MANUAL_ALIASES[ideal] in actual_headers
④ Fuzzy — token_sort_ratio(ideal, actual) ≥ threshold
Known unmatched by design (fuzzy <80)
LastUpdateDate LastUpdateLogin LastUpdatedBy SourceOrganizationId (~0.72) SourceSubinventory (~0.70)
Imports
aliases.MANUAL_ALIASES normalizer.strip_header types.MappingResult rapidfuzz
⚖
comparator.py
engine/comparator.py
+
Deterministic BU x config classification. Iterates every (BU, matched config) pair and emits rows into three buckets: CIP, gaps, additional.
Main function
compare(mapping, actual_df, ideal_df) → (cip_list, gaps_list, additional_list)
Classification logic (per BU x config)
is_capture(ideal_val) → additional (extract only, no comparison)
norm(actual) == norm(ideal) → CIP (control in place)
actual is null, ideal not capture → Control gap
norm(actual) != norm(ideal) → gap
Imports
normalizer.normalize_value normalizer.is_capture types.AuditRow

⚙
audit_worker.py
app/workers/audit_worker.py
+
RQ job executor. Bridge between async web layer and synchronous engine. Runs full pipeline in a worker process. Updates job status on success or failure.
Function
run_audit_job(job_id: str) — sync, called by RQ
Execution sequence
1. Load job via SyncSessionLocal
2. ideals_loader.get_ideal_bytes(check_id, tenant_id)
3. engine.read_actual + engine.read_ideal
4. engine.build_mapping(ideal_rows, actual_headers)
5. engine.compare(mapping, actual_df, ideal_df)
6. engine.build_report(cip, gaps, additional, mapping)
7. storage.save(report_key, report_bytes)
8. UPDATE AuditJob status=done (or failed + error_message)
