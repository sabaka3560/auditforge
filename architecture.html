Engine Modules — click to expand
engine/types.py
Shared Types
Shared dataclasses. No logic, no I/O. Everything else imports from here.
@dataclass MappingResult @dataclass AuditRow

Why separate?

Prevents circular imports. matcher.py and comparator.py both need these types but must not import each other.

Rule

Never add logic here. If you need a method on a dataclass, write a standalone function in the relevant module.

engine/aliases.py
Shared Config
One dict. Oracle Fusion exports use different column names than ITAC ideal files use. This maps them.
MANUAL_ALIASES: dict[str, str]

Current aliases (7)

AllowItemSubstitutionsFlag → AllowItemSubstitutions BusinessUnitId → BusinessUnitName MasterOrganizationId → MasterOrgCode ProfitCenterBuId → ProfitCenterBuName SupplierId → SupplierName SupplierSiteId → SupplierSiteName OrganizationId → OrganizationCode

How to extend

Add to MANUAL_ALIASES and write a test in test_matcher.py. No other file changes needed.

engine/normalizer.py
Normalization
All string normalization lives here. strip_header for headers, normalize_value for comparison, is_capture for capture detection.
strip_header(s) → str normalize_value(v) → str is_capture(ideal) → bool detect_ideal_columns(df) → (str, str)

Boolean synonym map

Y / y / yes / Yes / YES / true / 1 / enabled / on → "y" N / n / no / No / NO / false/ 0 / disabled/ off → "n"

Capture phrases

"capture", "record the value", "extract", "document the actual", "note the value", "for information only", "informational"
engine/reader.py
Ingestion
File I/O only. Reads client file and ideal value file into DataFrames. No comparison, no matching.
read_actual(file_obj, sheet) → DataFrame read_ideal(file_obj) → DataFrame

read_actual

  • Raises ValueError if sheet_name not found (returns available sheets)
  • Renames column A to BU_NAME regardless of original header
  • Drops rows where BU_NAME is blank or NaN

read_ideal

  • Tries openpyxl first, falls back to CSV parser
  • Auto-detects name/value columns via normalizer.detect_ideal_columns()
  • Returns exactly 2 columns: config_name, ideal_value
engine/matcher.py
Matching
Resolves each ideal config name to an actual column header using the 4-stage waterfall. Returns one MappingResult per ideal row.
build_mapping(ideal_df, headers, threshold) → list[MappingResult]

Internal functions

_match_one() — runs the 4 stages for a single ideal name _best_fuzzy() — scans all actuals, returns (score, header)

Fuzzy scorer

Uses token_sort_ratio, not ratio. ERP config names are token bags — AllowItemSubstitutionsFlag vs AllowItemSubstitutions works correctly with token_sort but fails with plain ratio.

engine/comparator.py
Comparison
The only module that touches actual cell values. For each BU × matched config: capture check → normalize → compare → classify.
compare(actual_df, mapping) → (cip, gaps, extra)

Classification order

1. is_capture(ideal_value) → additional data 2. normalize(actual) == normalize(ideal) → controls in place 3. else → control gaps

Null handling

NaN/None actual → empty string after normalize_value(). Empty string ≠ any ideal value → control gap.

engine/styles.py
Output
openpyxl style constants only. Change colors in one place.
HEADER_FILL — #1F4E79 dark blue GREEN_FILL — #C6EFCE controls in place RED_FILL — #FFC7CE control gaps BLUE_FILL — #DDEBF7 additional data YELLOW_FILL — #FFEB9C unmatched rows

Why separate?

writer.py imports from here. Centralizing styles means you change colors in one file, not scattered through sheet-writing code.

engine/writer.py
Output
Builds the 5-sheet Excel workbook entirely in memory. Returns bytes — nothing touches disk.
build_report(cip, gaps, extra, mapping, ...) → bytes

Why openpyxl over pandas.to_excel?

pandas.to_excel gives no per-cell styling control. Audit outputs require color-coded rows (green/red/blue), styled headers, and auto-fit columns — all requiring openpyxl PatternFill and Font APIs directly.

Internal helpers

_data_sheet() — writes Controls in place / gaps / extra _mapping_sheet() — writes Header Mapping and Exc _summary_sheet() — writes Audit Summary _header_row() — styled header (shared) _fill_row() — applies fill to current row _autofit() — auto-fit column widths

Karpathy Rules — Applied to This Codebase
01
Read before writeNever edit a file without reading it. Always read the diff before committing.
02
Stay close to the dataRun the tool against real Excel files after every non-trivial change. Unit tests on toy data miss real bugs.
03
One file, one purposereader.py reads files. matcher.py matches. Nothing else. Parsing logic in comparator.py is a bug.
04
Explicit over implicitNo magic, no metaprogramming. If a function surprises on re-read, simplify it.
05
Functions should be boringFits on a screen. Typed input/output. No side effects. That's a good function.
06
Delete code as a featureThe best refactor removes lines. If you added something "just in case", remove it.
07
Test with production-like dataTests use real Oracle Fusion BU names and config patterns. Don't replace with foo/bar.
08
Commit after each green testSmall commits are reviewable. Large commits are not.
09
Type hints are not optionalEvery public function must be typed. Any is a code smell.
10
Comments explain WHYThe code says what it does. Comments explain why a surprising decision was made. Obvious things need no comments.
11
No global mutable stateAll state flows through function parameters. Nothing shared between requests.
12
The diff is the productEvery PR reviewable in under 5 minutes. If it isn't, it's too big.
